Jumping Box

Help a little box jump over obstacles and see how far you can go. The more obstacles you pass, the higher your score. Be careful! One collision and it's game over.


How it works

The game starts with a box on the ground and an obstacle moving towards it. The box can jump to avoid the obstacle. When the box collides with the obstacle, the game ends and the final score is displayed on the screen.

Getting Started

Follow these steps to build the game. First set up a new file and import the relevant modules. Then draw the Actors and define the functions to run the game. Good luck!

Open Replit

Create a new Repl called jumping_box.

Set up an image folder

This game uses images of a box and an obstacle. In the upper left corner of the screen, click the folder icon with a plus sign and name the folder images.

Put the images into the folder

Click HERE to create the Jumping Box images. Save your box (or character) and obstacle as "box.png" and "obstacle.png". Click the three dots and select Upload file. Then, select box.png and obstacle.png from the files.

Drag box.png and obstacle.png into the images folder. Your folders should look something like this example.

Import pgzrun

Remember, in order to run our game, we need to import pgzrun. Add the below code. 
import pgzrun

Import random

Since we will need to generate random positions for the obstacles, we also need to import the random library. Add the following line of code. 
import random

Set the game variables

Set up the game variables such as screen width and height, obstacle speed, jump height, and the initial score. Type this code at the top of your file. 

    WIDTH = 700
    HEIGHT = 400
    OBSTACLE_SPEED = 5
    JUMP_HEIGHT = 240
    GRAVITY = 4
    SCORE = 0

Create the Actors

Now, let’s create the Actors—a box and an obstacle. To create them and set their positions, add these lines of code under the game variables. 

    box = Actor("box", (100, HEIGHT - 25))
    obstacle = Actor("obstacle", (WIDTH, HEIGHT - 25))

Set up game states

We need two Boolean variables (variables whose values can either be True or False) to tell Pygame Zero if the game is over or if the start screen is displayed. At this stage, set both variables to False--this means the game is NOT over and the start screen is NOT displayed when we begin. 

    game_over = False
    start_screen = True

Draw the game elements

Now you need to use the draw() function to display the Actors on the screen, change the background color, and display the score.

First, we define our draw() function. 

    def draw():
Then, we check if the start_screen variable is True, which means we are at the starting screen of the game. If start_screen is True, we fill the entire screen with a green color as the background for the start screen and draw the text "Press Enter to Begin" at the center of the screen with a font size of 60 and a white color. 

        if start_screen:
            screen.fill("Green")
            screen.draw.text("Press Enter to Begin",
                             center=(WIDTH // 2, HEIGHT // 2),
                             fontsize=60,
                             color="white")
We check if the game_over variable is True, which means the game has ended. If game_over is True, we fill the entire screen with an orange color as the background for the game over screen and draw the text "GAME OVER" at the center of the screen with a font size of 60 and a white color. 
    
        elif game_over:
            screen.fill("orange")
            screen.draw.text("GAME OVER",
                             center=(WIDTH // 2, HEIGHT // 2),
                             fontsize=60,
                             color="white")
This part of the code runs when both start_screen and game_over are False, which means the game is currently being played. We clear the screen of any previous drawings. We then fill the entire screen with a brown color as the background for the gameplay. After that, we draw the box and obstacle Actors on the screen at their current positions. Finally, we draw the current score at the top-left corner of the screen with a font size of 30 and a white color. 

        else:
            screen.clear()
            screen.fill("brown")
            box.draw()
            obstacle.draw()
            screen.draw.text("Score: " + str(SCORE), (10, 10),
                           fontsize=30,
                           color="white")

Update the game

The update() function handles the game logic, such as obstacle movement, box jumping, and collision detection. Type in the following code to define the update() function.

Define the update() function. 

    def update():
Write these two lines declare the global variables SCORE, start_screen, and game_over. This allows us to access and modify them within the update() function. 

        global SCORE
        global start_screen, game_over
Write this block of code to check if the game is in the start_screen or game_over state. If either of these is True, the function immediately returns, and the rest of the code in the update() function is not executed. This keeps the game from starting before it's supposed to 

        if start_screen or game_over:
            return
Write these lines handle the movement of the obstacle. The obstacle's x-coordinate is reduced by OBSTACLE_SPEED, moving it to the left. If the right side of the obstacle goes off-screen (its x-coordinate is less than 0), the reset_obstacle() function is called, and the SCORE is increased by 1. 

        obstacle.x -= OBSTACLE_SPEED
        if obstacle.right < 0:
            reset_obstacle()
            SCORE += 1
Write this block of code to control the box's jumping action. If the up arrow key is pressed and the box is on the ground (its y-coordinate is equal to `HEIGHT - 25`), the box's y-coordinate is reduced by JUMP_HEIGHT, making it jump. 

        if keyboard.up and box.y == HEIGHT - 25:
            box.y -= JUMP_HEIGHT
Write this code to handle the box's falling action. If the box is not on the ground (its y-coordinate is less than `HEIGHT - 25`), the box's y-coordinate is increased by GRAVITY, causing it to fall. 

        if box.y < HEIGHT - 25:
            box.y += GRAVITY
Write this block to check for collisions between the box and the obstacle using the colliderect() method. If a collision occurs, the game_over variable is set to True, ending the game. 

        if box.colliderect(obstacle):
            game_over = True

Handle key presses

Now, let's define the on_key_down() function that will handle the Enter key press to start the game. Add the following code. 

        def on_key_down(key):
            if key == keys.RETURN:
                global start_screen
                start_screen = False

Reset the obstacle

Define the reset_obstacle() function that will reset the obstacle position once it goes off-screen. Type in the following code. 

        def reset_obstacle():
            obstacle.x = WIDTH

Call pgzrun.go()

In order to start the game, we have to use our pgzrun import by calling pgzrun.go() at the very end of our program. Add the following line to your program. MAKE SURE IT'S ALWAYS THE LAST LINE OF CODE IN YOUR PROGRAM. 
pgzrun.go()

Try it out

Now test the code you’ve written so far. Go ahead and click Run and see what happens. Did your game run? You should see the start screen with "Press Enter to Begin" text. Press Enter to start the game, and try to jump over the obstacles. Check if the game ends when the box collides with the obstacle.

Hacks and Tweaks

Additionally, you can try the following customizations to make your game even more interesting:

Change the box and obstacle images

You can replace the box and obstacle images with other characters or objects by using different images. Remember to update the code so it uses the names of the new images throughout the program.

Adjust the game speed

The game currently has a fixed obstacle speed. To make the game more challenging, you could increase the speed of the obstacles as the score increases. You can do this by updating the OBSTACLE_SPEED value based on the SCORE.

Change the playing area

You can change the size of the playing area by changing the values of WIDTH and HEIGHT. Try using different numbers for these values and see what happens.

Add Levels

Make the game harder as the score increases. You can try adding more obstacles, increasing the speed, or having the box move slower as the score and level increase.
